## 📋 Brief Summary

A ground-up explainer of the **vocabulary of AI-assisted development**: what an LLM actually does (predict the next token), what tokens are, how neural networks + Transformers make it work, where the "knowledge" comes from (training), why models "get dumber" in long sessions (the context window), and how plain text-predictors became **agents** that execute tasks — plus two of the best-known ecosystem concepts, **skills and MCPs**. The goal is a mental map of the whole AI-for-development ecosystem rather than a tutorial for any specific tool.

## 🎓 Mini-Lesson: The Concepts

### 1. What an LLM really does

- An LLM is a **function/box** that takes an incomplete text and returns **a list of probabilities for the next token**. "The capital of France is…" → París 72%, Lyon 10%, Marsella 7%, Roma 3%.
- Run that **in a loop** and you generate text indefinitely — that's literally how chats like ChatGPT work.

### 2. Tokens

- The **minimum processing unit** of an LLM — and the unit you *pay* in.
- Roughly ≈ one word, but not always: a word can split into several tokens (the video shows it in OpenAI's *Tokenizer* tool: "la capital de Francia" = 5 tokens, one per word — but "suscríbete" alone = 4 tokens).
- Internally tokens are **numbers (token IDs)** — these models are mathematics; text must be translated to numbers first.

### 3. Neural networks → Transformers

- A **neural network** is a math model inspired by brain neurons: input layer → computation layers → prediction. Classic uses: handwriting recognition (OCR), classification.
- The video demos an interactive NN playground (link in the description): load spiral data, press play, and watch the network learn **colored decision regions** — a new point landing in the blue zone gets classified blue, in the orange zone orange.
- **Limitation:** long-distance relations in language. In "the cat didn't cross the street because it was tired", *cat* and *tired* are far apart — classic networks struggled to link them (early GPTs produced grammatical nonsense).
- **2017, Google's "Attention Is All You Need" paper → the Transformer architecture:** assigns a **weight to the relation of every token with every other token** (cansado↔gato: 65%). Networks "vitaminized" with Transformers see whole sentences and their internal relevance — the key to coherent generation.
- (Recommended deep-dive: 3Blue1Brown's visual videos on the topic.)

### 4. Training — where the "knowledge" comes from

- If LLMs only predict tokens, why do they "know" Paris? **Training.**
- You feed a **training set** matched to the goal (to write text: books, scripts, web articles — companies famously used the Internet).
- Mechanism (Harry Potter example): chop texts, hide the last word, make the model predict it, compare with the original, **adjust the weights** on error, repeat — until the configuration converges toward the goal.
- This is also where **biases** come from, and why models differ (architecture *and* training data). Fun aside: some models have reportedly been trained on text generated by *other* AIs (e.g., Claude outputs).

### 5. The context window — why models "get dumber"

- Everything you send (and everything an agent reads) consumes the **context window**, which is finite.
- When the limit approaches, models cope differently:
  - Simple: **drop the oldest tokens** (it literally forgets).
  - Smarter: **summarize** old tokens into a cheaper representation.
  - Others: weight recent tokens above old ones (a gradient).
- As a developer you must **manage what goes into context** — it directly determines answer quality. (This becomes the core theme of harness engineering → see the harness summary.)

### 6. Agents — from text prediction to action

- The chat pattern: prompt in → tokens out, with conversation history as context.
- An **agent** changes the game: a program that **perceives its environment and executes actions**, using the LLM as its *brain*. The loop: **perceive → think → act**.
- Example: Claude Code — "run this project's tests" doesn't just generate text; it decides, asks permission, executes scripts, reads the filesystem.
- ⚠️ Every file read and every command output **also fills the context** — the more an agent reads, the sooner it forgets things.

### 7. Skills and MCPs

Two of the best-known ecosystem concepts for organizing agent capabilities:

- **Skill** = a *capability* you give the agent (search the web, query a DB, get the weather) — knowledge packaged so the model **pulls it into context dynamically, only when needed**, instead of bloating every prompt. (Helps context management: why carry DB instructions if you'll never call the DB?) In Claude Code: `/skills`, stored in a folder.
- **MCP (Model Context Protocol)** = the **standard** for connecting AIs to traditional software tools — the same role REST/GraphQL play for web APIs. The skill explains *how*; the MCP provides *the door*. In Claude Code: `/mcp` (e.g., Gmail, Google Calendar servers).

### 8. Closing advice: keep distance from the hype

- Two-three years ago we argued about editors; now about models and skills. The methodologies are **still settling** — every few months "the way to program" changes (autocomplete → CLIs like Claude Code → 10 parallel agents).
- Recommendation: stay informed, but **with distance** — adopt what genuinely interests you, or you'll be switching all your tools every two months.
- One methodology that *does* seem to be settling: **Spec-Driven Development** (→ see the SDD summary).

## 🧭 The Big Picture (mental map)

```
  TEXT ──tokenizer──► TOKENS (≈words; really numbers/IDs; the billing unit)
                          │
        NEURAL NETWORK + TRANSFORMERS ("Attention Is All You Need", 2017)
        weights between every token pair → understands long-range relations
                          │
        TRAINING: predict hidden word → compare → adjust weights → repeat
        (training set ⇒ knowledge AND biases; differs per company/model)
                          │
        LLM = next-token probability function, looped = text generation
                          │
        CONTEXT WINDOW (finite!): drop-oldest / summarize / recency-weight
        → "the model got dumber" = context filled up; manage it deliberately
                          │
        AGENT = perceive → think (LLM as brain) → act (tools, scripts, files)
        every action's output ALSO fills the context
                          │
        SKILLS (capabilities, loaded on demand)  +  MCP (the standard
        connector between AIs and software — the REST of the AI world)
```

---

## 💡 Key Takeaways

1. **An LLM is a next-token probability function in a loop** — the "knowledge" is trained weights, not understanding.
2. **Transformers (2017) made coherence possible** by weighting relations between all tokens in a sentence.
3. **The context window is the scarce resource** — long sessions degrade because context fills; everything an agent reads counts.
4. **Agents = LLM brain + perceive/think/act loop** — that's the jump from generating text to executing tasks.
5. **Skills give capabilities on demand; MCP standardizes the connection to tools** — both are also context-management mechanisms.
